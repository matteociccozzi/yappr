yappr — Open-Source Hardening Plan

Generated from a live fresh-install audit + deep repo analysis. Goal: git clone --recurse-submodules … && ./scripts/install.sh works for anyone, the repo feels like a proper Unix CLI tool, and contributors can navigate and extend it.

Tier 1 · Fix install · 7 tasks · ~1 day
Tier 2 · Runtime separation · 6 tasks · ~2 days
Tier 3 · Polish + OSS readiness · 11 tasks · ~4 days
Tier 4 · Documentation · 11 tasks · ~1 day

Cross-cutting decisions — XDG path conventions

All env vars follow XDG with macOS-compatible defaults. Single source of truth: bin/_yappr-paths.sh (bash) and bin/_yappr_paths.py (Python). Swift daemon reads the same vars via ProcessInfo.processInfo.environment.

LogicalEnv varDefault
Repo / source rootYAPPR_ROOTself-detected via dirname "${BASH_SOURCE[0]}"
Config dirYAPPR_CONFIG_HOME${XDG_CONFIG_HOME:-$HOME/.config}/yappr
Data dirYAPPR_DATA_HOME${XDG_DATA_HOME:-$HOME/.local/share}/yappr
State dirYAPPR_STATE_HOME${XDG_STATE_HOME:-$HOME/.local/state}/yappr
Cache dirYAPPR_CACHE_HOME${XDG_CACHE_HOME:-$HOME/.cache}/yappr
Runtime dirYAPPR_RUNTIME_DIR${XDG_RUNTIME_DIR:-/tmp/yappr-$(id -u)}
SocketYAPPR_SOCKET$YAPPR_RUNTIME_DIR/stt.sock
Trace logYAPPR_TRACE_LOG$YAPPR_RUNTIME_DIR/trace.log
Daemon logYAPPR_DAEMON_LOG$YAPPR_STATE_HOME/logs/daemon.log
Daemon PIDYAPPR_DAEMON_PID$YAPPR_RUNTIME_DIR/daemon.pid

1.3 Auto-populate Nemotron model cache during install M ›
Depends on: 1.1
Problem
First daemon launch fails with CoreML "model not found at ~/.cache/fluidaudio/…". Fix requires building fluidaudiocli from vendor/FluidAudio and running it against a silent WAV — none of this is automated or documented.
Files to change
  • scripts/install.sh — add step after Swift daemon build, before codesign:
step "Populate Nemotron model cache (~/.cache/fluidaudio/)"
CACHE_DIR="$HOME/.cache/fluidaudio/models/nemotron-streaming/560ms"
if [[ -f "$CACHE_DIR/preprocessor.mlmodelc/coremldata.bin" ]]; then
  ok "models already cached"
else
  info "Building fluidaudiocli and downloading Nemotron models (~200 MB)..."
  (cd "$YAPPR_ROOT/vendor/FluidAudio" \
    && swift build -c release --product fluidaudiocli \
                   --scratch-path "$YAPPR_DATA_HOME/build/fluidaudio")
  WARMUP="$(mktemp).wav"
  python3 -c "
import wave, struct
with wave.open('$WARMUP', 'w') as f:
    f.setnchannels(1); f.setsampwidth(2); f.setframerate(16000)
    f.writeframes(struct.pack('<'+'h'*16000, *([0]*16000)))
"
  "$YAPPR_DATA_HOME/build/fluidaudio/release/fluidaudiocli" \
    nemotron-transcribe --input "$WARMUP" --chunk 560 >/dev/null
  rm -f "$WARMUP"
  [[ -f "$CACHE_DIR/preprocessor.mlmodelc/coremldata.bin" ]] \
    || fail "model cache still empty after warmup"
  ok "models cached at $CACHE_DIR"
fi

Tier 2

Separate source from runtime data

Goal: source tree contains only code + shipped configs + docs. All writes go to XDG dirs. git status is clean after install and after daily use. git pull never fights user data.
2.1 Introduce path-resolution helpers M ›

Foundational — everything else in Tier 2 and 3 depends on this.

Files to create
  • bin/_yappr-paths.sh — sourced by all bash scripts. Sets and exports all YAPPR_* env vars. Provides: yappr_ensure_dirs, yappr_log_path, yappr_metric_path, yappr_connect_binary.
  • bin/_yappr_paths.py — importable Python module with identical constants. Provides: paths.runtime_dir(), paths.metrics_dir(), paths.logs_dir(), paths.trace_log(), paths.socket(), paths.daemon_log(), paths.config_dir().
# bin/_yappr-paths.sh (excerpt)
YAPPR_ROOT="${YAPPR_ROOT:-$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)}"
YAPPR_CONFIG_HOME="${YAPPR_CONFIG_HOME:-${XDG_CONFIG_HOME:-$HOME/.config}/yappr}"
YAPPR_DATA_HOME="${YAPPR_DATA_HOME:-${XDG_DATA_HOME:-$HOME/.local/share}/yappr}"
YAPPR_STATE_HOME="${YAPPR_STATE_HOME:-${XDG_STATE_HOME:-$HOME/.local/state}/yappr}"
YAPPR_RUNTIME_DIR="${YAPPR_RUNTIME_DIR:-${XDG_RUNTIME_DIR:-/tmp/yappr-$(id -u)}}"
YAPPR_SOCKET="${YAPPR_SOCKET:-$YAPPR_RUNTIME_DIR/stt.sock}"
YAPPR_TRACE_LOG="${YAPPR_TRACE_LOG:-$YAPPR_RUNTIME_DIR/trace.log}"
export YAPPR_ROOT YAPPR_CONFIG_HOME YAPPR_DATA_HOME YAPPR_STATE_HOME
export YAPPR_RUNTIME_DIR YAPPR_SOCKET YAPPR_TRACE_LOG

2.4 Move socket + trace + daemon log + PID to $YAPPR_RUNTIME_DIR L ›
Depends on: 2.1

Eliminates all hardcoded /tmp/yappr-* paths from Swift source. Makes multi-user installs safe.

Files to change
  • swift/…/Daemon.swift — socketPath reads YAPPR_SOCKET env var, falls back to $YAPPR_RUNTIME_DIR/stt.sock. Creates runtime dir before bind.
  • swift/…/Trace.swift — trace log path reads YAPPR_TRACE_LOG env var.
  • swift/…/YapprSttConnect/main.swift — socket and trace paths from env, same fallback.
  • bin/yappr — socket existence check uses $YAPPR_SOCKET.
  • bin/yappr-trace — LOG_PATH = paths.trace_log().
  • scripts/templates/hammerspoon-init.lua.tmpl — add @YAPPR_TRACE_LOG@ placeholder.
// Daemon.swift
static var socketPath: String {
    let env = ProcessInfo.processInfo.environment
    if let s = env["YAPPR_SOCKET"] { return s }
    let runtime = env["YAPPR_RUNTIME_DIR"] ?? "/tmp/yappr-\(getuid())"
    return "\(runtime)/stt.sock"
}

3.2 yappr becomes a subcommand dispatcher M ›
Depends on: 2.1
Files to change
  • Rename current bin/yappr → bin/yappr-dictate (orchestrator logic unchanged)
  • Create new bin/yappr dispatcher — Hammerspoon still spawns yappr with no args → routes to yappr-dictate, no init.lua changes needed
#!/usr/bin/env bash
source "$(dirname "${BASH_SOURCE[0]}")/_yappr-paths.sh"
case "${1:-dictate}" in
  dictate|"") exec "$YAPPR_ROOT/bin/yappr-dictate" "${@:2}" ;;
  daemon)     exec "$YAPPR_ROOT/bin/yappr-daemon"  "${@:2}" ;;
  config)     exec "$YAPPR_ROOT/bin/yappr-config"  "${@:2}" ;;
  stats)      exec "$YAPPR_ROOT/bin/yappr-stats"   "${@:2}" ;;
  trace)      exec "$YAPPR_ROOT/bin/yappr-trace"   "${@:2}" ;;
  doctor)     exec "$YAPPR_ROOT/bin/yappr-doctor"  "${@:2}" ;;
  server)     exec "$YAPPR_ROOT/bin/yappr-server"  "${@:2}" ;;
  help|-h|--help) exec "$YAPPR_ROOT/bin/yappr-help" ;;
  version|--version) echo "yappr $(cat "$YAPPR_ROOT/VERSION")"; exit 0 ;;
  *) echo "yappr: unknown subcommand '$1'" >&2; exit 2 ;;
esac
3.3 yappr daemon subcommand (start/stop/status/restart/logs/tail) M ›
Depends on: 3.1, 3.2
Create bin/yappr-daemon
  • start — check PID + kill -0; if not running → nohup $DAEMON_BIN >> $YAPPR_DAEMON_LOG 2>&1 &; write PID file; wait up to 5s for socket to appear
  • stop — read PID file, kill -TERM, wait 2s, escalate to kill -KILL, unlink socket and PID file
  • restart — stop + start
  • status — PID file + kill -0 + socket exists → print running (pid=X, socket OK) or not running
  • logs — cat $YAPPR_DAEMON_LOG
  • tail — tail -F $YAPPR_DAEMON_LOG
3.4 Daemon writes and cleans up a PID file S ›
Depends on: 2.4

let pidPath = ProcessInfo.processInfo.environment["YAPPR_DAEMON_PID"]
    ?? "\(runtimeDir)/daemon.pid"
try? "\(getpid())".write(toFile: pidPath, atomically: true, encoding: .utf8)
defer { try? FileManager.default.removeItem(atPath: pidPath) }
3.5 yappr doctor — post-install verifier L ›
Depends on: 2.1–2.4, 3.2

Python script. Prints [OK] / [WARN] / [FAIL] per check. Exits 1 on any FAIL. Each FAIL includes an explicit hint pointing to the relevant System Settings pane.

Checks (in order)
  • macOS + Apple Silicon arch
  • Required commands on PATH: jq, python3
  • Required dirs exist + writable: $YAPPR_CONFIG_HOME, $YAPPR_STATE_HOME/logs, $YAPPR_STATE_HOME/metrics, $YAPPR_RUNTIME_DIR
  • Active config: valid JSON, .llm.url parseable, .prompt_file resolves
  • Daemon binary exists + is codesigned (codesign -dv exits 0)
  • Daemon process running (PID file + kill -0)
  • Socket exists + connectable (connect + immediate close)
  • LLM endpoint reachable (GET /v1/models, max 2s timeout)
  • Nemotron model cache populated
  • Hammerspoon installed + init.lua references correct path
  • mlx_lm.server on PATH (warn-only — only needed for local LLM)
